# Recruiter Summary

**LLM-Security-Guardrails-Lab** is a portfolio-grade AI security lab for deterministic prompt-risk inspection and redaction examples.

## One-sentence summary

A small, testable LLM security baseline that inspects prompt-risk indicators, redacts common sensitive-looking values and validates guardrail behaviour through repeatable tests.

## What it proves

- Understanding of prompt-injection risk patterns.
- Practical data-minimisation and redaction thinking.
- Ability to build deterministic security checks.
- Honest limitation-setting: pattern-based guardrails are not complete protection.
- Testable Python implementation with CI.

## Best-fit roles

- AI Security Engineer
- LLM Application Security Analyst
- Responsible AI / AI Governance Analyst
- Cybersecurity GRC Analyst
- Security Automation Developer

## Interview talking points

1. Why deterministic guardrails are useful but incomplete.
2. How sensitive-value redaction supports safer LLM workflows.
3. Why prompt-injection detection needs layered controls.
4. How the lab could expand to output filtering, datasets and scoring.
5. How this connects to secure RAG and AI governance work.

## Current maturity

This is a clear baseline lab. Future improvements could add larger test datasets, indirect prompt-injection examples, scoring, output filtering and OWASP LLM Top 10 mapping.
