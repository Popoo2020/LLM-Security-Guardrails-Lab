# Hiring Manager Summary

## What this project demonstrates

LLM-Security-Guardrails-Lab demonstrates a compact, testable baseline for prompt-risk inspection and pre-processing before LLM use.

The project intentionally avoids overstating pattern-based guardrails. It presents a transparent baseline that can be tested, measured and improved.

## Relevant roles

- AI Security Analyst
- Responsible AI / AI Governance roles
- Cybersecurity GRC roles involving AI controls
- Secure AI application reviewer
- Junior security automation / Python portfolio review

## What this demonstrates

- Prompt-injection indicator awareness.
- Sensitive-value redaction examples.
- Deterministic tests instead of vague safety claims.
- Clear limitations around pattern-based detection.
- CI validation, linting and dependency audit.

## Current limitations

The current lab is rule-based and intentionally small. It does not claim to be a production LLM firewall and does not yet include model-aware evaluation, scoring metrics or large benchmark datasets.
